# DNS Server

## Сборка

Проект собирается с помощью CMake. Требуется компилятор с поддержкой C++20.

```bash
# Конфигурация и сборка
cmake -B build -S .
cmake --build build
```

## Тестирование

Все команды тестирования должны выполняться из директории `dns-server/build`.

### 1. Тест на производительность (Stress Test)

C++ стресс-тест позволяет оценить внутреннее масштабирование сервера при изменении количества потоков в `config.ini`, избегая накладных расходов ОС, свойственных скриптам с вызовом `dig`.

Сборка теста происходит автоматически. Запуск стресс-теста:

```bash
./build/stress_test 127.0.0.1 5353 4 10
```
(Параметры: IP, Порт, Потоки клиента, Продолжительность в секундах)

_Ожидаемый результат_: Вывод количества отправленных запросов (qps) и полученных ответов (rps). Изменяя `ThreadCount` в `config.ini` и перезапуская сервер, можно наглядно увидеть изменение пропускной способности (rps).

### 1.1 Bash Stress Test (Legacy)

Оценка с помощью утилиты `dig`:

```bash
./tests/dig_stress_test.sh
```

_Ожидаемый результат_: Автоматический запуск сервера, выполнение 100 параллельных запросов (50% хитов, 50% новых NXDOMAIN записей). Вывод содержит итоговый RPS.

### 2. Тест на корректность завершения (Shutdown Test)

Проверка чистого выхода сервера по сигналу SIGINT:

```bash
./build/shutdown_test ./build/dns-server
```

_Ожидаемый результат_: Сообщение "SUCCESS: Clean shutdown is fast". Время завершения < 2 секунд.

### 3. Сценарии использования

Скрипт `test_queries.sh` для проверки типов записей (A, AAAA, CNAME) и негативного кэширования.
